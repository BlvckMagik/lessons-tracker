# Lesson Tracker

Застосунок для вчителів — трекінг уроків, учнів та оплат. Інтерактивний календар для планування розкладу, управління учнями, автоматичний розрахунок вартості та формування звітів по оплатах.

## Технічний стек

| Категорія | Технологія |
|-----------|-----------|
| Фреймворк | Next.js 15 (App Router) |
| UI | React 19, MUI v6 (темна тема) |
| Стейт | Redux Toolkit, RTK Query (optimistic updates) |
| Таблиці | TanStack React Table v8 |
| Календар | FullCalendar |
| База даних | Prisma + SQLite (файлова, персистентна) |
| Мова | TypeScript (strict) |
| Архітектура | Feature-Sliced Design (FSD) |

## Швидкий старт

```bash
# Встановлення залежностей
pnpm install

# Ініціалізація бази даних
pnpm db:push

# Запуск dev-сервера
pnpm dev
```

Застосунок буде доступний на [http://localhost:3000](http://localhost:3000).

## Функціонал

### Календар (головна сторінка)

- Тижневий, денний та місячний вигляд
- Створення уроку через виділення часового слоту
- Попап з деталями уроку при натисканні на подію
- Після закінчення уроку з'являються кнопки підтвердження статусу
- Колірна індикація статусів

### Учні (`/students`)

- Таблиця з пошуком та сортуванням
- Створення та видалення учнів
- Поля: ім'я, телефон, email

### Уроки

- **Тип:** індивідуальне або групове
- **Предмет:** англійська або німецька
- **Ціна:** індивідуальне — 200 грн, групове — 50 грн з кожного учня
- **Вибір учнів:** звичайний select для індивідуалки, мультиселект для групи

### Статуси уроків

| Статус | Нарахування |
|--------|-------------|
| Проведено | Оплату нараховано |
| Пропущено | Оплату нараховано |
| Скасовано | Оплату НЕ нараховано |

### Звіти (`/reports`)

- Зведена таблиця по кожному учню
- Кількість проведених, пропущених, скасованих уроків
- Загальна сума нарахувань, оплат та боргу
- Розгортання рядка — деталі кожного уроку з чекбоксом оплати
- Фільтрація та сортування

## Структура проекту (FSD)

```
src/
├── app/                  # Next.js App Router, layout, providers, API routes
├── widgets/              # Композитні UI-блоки (календар, таблиці, сайдбар)
├── features/             # Користувацькі взаємодії (створення уроку/учня, статуси)
├── entities/             # Бізнес-сутності (student, lesson) з RTK Query API
└── shared/               # Утиліти, базовий API, константи, Prisma client
```

### Правила імпорту між шарами

- `shared` → жодних імпортів з верхніх шарів
- `entities` → тільки `shared`
- `features` → `entities`, `shared`
- `widgets` → `features`, `entities`, `shared`
- `app` → всі шари

## База даних

SQLite — файлова база, зберігається у `prisma/dev.db`. Не зникає при перезапуску чи перезбірці.

### Схема

- **Student** — id, name, phone, email
- **Lesson** — id, type, subject, startTime, endTime, status, pricePerStudent
- **LessonStudent** — зв'язок many-to-many між уроком та учнем, поле `paid`

Для перегляду та редагування бази через GUI:

```bash
pnpm db:studio
```
